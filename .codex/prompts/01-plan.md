# Command: plan

Use when the user asks for a roadmap, new feature plan, or multi-step fix.

Format:
- PLAN: at least three numbered steps with target environment (dev|stage|prd|custom).
- SYNCHRONIZE: mention hooks/pre-tool-guard.md and mcp/mcp-standards.md if present.
- EXECUTE: list expected tools (Node, Python, Docker, GitHub Actions, AWS/VPS, Prometheus/Grafana, npm/pnpm, PyPI, MCP).
- OBSERVE: list how results will be validated.
- REFLECT: what the human should verify before proceeding.
