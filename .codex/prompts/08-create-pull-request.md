# Command: create-pull-request

Use to open a PR with a clear summary.

Workflow:
1. Verify branch and status; never on main/master.
2. Push branch and set upstream.
3. Summarize diff and risks.
4. Create PR with conventional title and short body.

PR body format:
- Summary (3-6 bullets)
- Type (feat/fix/refactor/docs/chore)
- Tests (if any)
